## 1. 테스트 환경 준비

### 백엔드 실행
```bash
cd backend
pip install -r requirements.txt   # 최초 1회
uvicorn app.main:app --reload
```

### 프론트엔드 실행
```bash
cd frontend
npm install                        # 최초 1회
npm run dev
```

### Ollama 실행 확인
```bash
ollama serve                       # 서버 미실행 시
ollama pull llama3                 # 모델 미설치 시
```

### 확인 URL

| 항목 | 주소 |
|------|------|
| 프론트엔드 | http://localhost:3000 |
| 백엔드 헬스체크 | http://localhost:8000/health |
| Swagger UI | http://localhost:8000/docs |

### B. 긴 문서 처리 (chunking)

> 기준: 1,500자 초과 시 chunking 동작, chunk당 목표 크기 1,000자

| ID | 테스트명 | 사전 조건 | 실행 방법 | 기대 결과 |
|----|---------|----------|----------|----------|
| B-1 | 긴 문서 chunking 동작 | Ollama 실행 중 | 1,500자 이상 텍스트(단락 구분 포함) 입력 → 요약하기 클릭 | steps에 "문서 분할 완료 (N개 chunk)", "chunk N/N 요약 완료", "최종 요약 생성 완료" 순서로 표시 |
| B-2 | 단락 없는 긴 텍스트 | Ollama 실행 중 | 줄바꿈 없이 1,500자 이상 텍스트 입력 → 요약하기 클릭 | 문장 단위 또는 강제 절단으로 분할되어 정상 요약 완료 |
| B-3 | 최대 chunk 초과 | - | chunk가 11개 이상 생성될 만큼 매우 긴 텍스트 입력 | 에러 메시지: "최대 허용 chunk 수(10개)를 초과" |

> **B-1 테스트용 샘플**: 아래 텍스트를 복사해 3회 이상 이어붙이면 1,500자 초과됩니다.
> ```
> 인공지능은 컴퓨터 과학의 한 분야로, 기계가 인간의 지능적 행동을 모방하도록 하는 기술입니다.
> 머신러닝은 인공지능의 하위 분야로, 데이터로부터 패턴을 학습합니다.
> 딥러닝은 신경망을 여러 층으로 쌓아 복잡한 문제를 해결합니다.
> ```

### C. txt 파일 업로드

> 테스트용 파일 준비 방법: 메모장 또는 텍스트 편집기로 직접 생성합니다.

| ID | 테스트명 | 사전 조건 | 실행 방법 | 기대 결과 |
|----|---------|----------|----------|----------|
| C-1 | 정상 txt 파일 업로드 | Ollama 실행 중, 10자 이상 내용의 txt 파일 | 파일 선택 → 요약하기 클릭 | 요약 성공, steps 첫 두 항목이 "파일 수신 완료", "텍스트 추출 완료" |
| C-2 | UTF-8 BOM 포함 txt | Ollama 실행 중 | Windows 메모장으로 "UTF-8 BOM" 저장한 txt 업로드 → 요약하기 클릭 | 정상 요약 (BOM 문자 없이 내용만 처리) |
| C-3 | 빈 txt 파일 | - | 내용이 없는 빈 txt 파일 업로드 → 요약하기 클릭 | 에러 메시지: "파일 내용이 비어 있습니다" |
| C-4 | 10자 미만 txt 파일 | - | `안녕` 만 적힌 txt 파일 업로드 → 요약하기 클릭 | 에러 메시지: "텍스트가 너무 짧습니다" |
| C-5 | txt가 아닌 파일 | - | `.pdf` 또는 `.docx` 파일 업로드 시도 | 파일 선택창에서 차단 (accept=".txt"), 우회 시 에러 메시지: "txt 파일만 업로드할 수 있습니다" |
| C-6 | EUC-KR 인코딩 txt | - | EUC-KR로 저장한 txt 파일 업로드 → 요약하기 클릭 | 에러 메시지: "파일을 UTF-8로 읽을 수 없습니다" |

> **C-2 파일 만들기**: Windows 메모장 → 다른 이름으로 저장 → 인코딩: `UTF-8 BOM` 선택 → 저장
